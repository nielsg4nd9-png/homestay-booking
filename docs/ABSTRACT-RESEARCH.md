# บทคัดย่อ (รูปแบบวิจัย 5 บท)

## ระบบจองที่พักโฮมสเตย์ออนไลน์

---

### บทที่ 1 บทนำ

**ภูมิหลังและความสำคัญของปัญหา**  
โฮมสเตย์และที่พักขนาดเล็กจำนวนมากยังใช้การจองผ่านโทรศัพท์หรือแชทซึ่งทำให้เกิดความคลาดเคลื่อน การจองซ้ำซ้อน และการติดตามมัดจำ/ยอดชำระทำได้ยาก จึงมีความต้องการระบบจองที่พักออนไลน์ที่รองรับการลงทะเบียนผู้ใช้ การเลือกห้องและช่วงวันที่ การคำนวณราคาและมัดจำ การอัปโหลดหลักฐานการโอน และการจัดการรายการจองรวมถึงการตรวจสอบมัดจำจากฝั่งผู้ดูแล

**วัตถุประสงค์ของการพัฒนา**  
1) พัฒนาระบบจองที่พักโฮมสเตย์บนเว็บที่ให้ผู้ใช้สมัครสมาชิก ล็อกอิน ดูห้องพักและราคา จองห้อง และอัปโหลดสลิปมัดจำ 2) ให้ผู้ดูแลระบบสามารถจัดการห้องพัก รายการจอง สินค้า/บริการ สไลด์หน้าแรก และผู้ใช้ รวมถึงตรวจสอบและอนุมัติ/ปฏิเสธหลักฐานการชำระมัดจำ

**ขอบเขต**  
ระบบเป็นเว็บแอปพลิเคชันสำหรับจองห้องพักหนึ่งแห่ง ใช้การชำระมัดจำ 50% และคงเหลือ 50% รองรับการอัปโหลดสลิปการโอนและให้แอดมินตรวจสอบ ไม่รวมการชำระเงินออนไลน์แบบ real-time (เช่น payment gateway) และไม่รวมการจองหลายธุรกิจในระบบเดียว

---

### บทที่ 2 วรรณกรรมและงานที่เกี่ยวข้อง

**แนวคิดและเทคโนโลยีที่ใช้**  
ระบบใช้สถาปัตยกรรมเว็บแบบ SPA (Single Page Application) บนเฟรมเวิร์ก Next.js (App Router) กับภาษา TypeScript เพื่อให้ได้ทั้งการแสดงผลฝั่งเซิร์ฟเวอร์และประสบการณ์ผู้ใช้แบบไดนามิก ใช้ Prisma เป็น ORM กับฐานข้อมูล SQLite สำหรับจัดเก็บข้อมูลผู้ใช้ ห้องพัก การจอง สินค้า/บริการ และสไลด์หน้าแรก การยืนยันตัวตนใช้ NextAuth (Credentials Provider และ JWT) แยกบทบาทผู้ใช้เป็น ADMIN, EMPLOYEE และ USER

**งานที่เกี่ยวข้อง**  
การจองที่พักออนไลน์โดยทั่วไปประกอบด้วย กระบวนการลงทะเบียน/ล็อกอิน การค้นหาและเลือกห้อง การกำหนดวันเช็คอิน–เช็คเอาท์ การคำนวณราคาและมัดจำ การยืนยันการจอง และการชำระเงิน/มัดจำ ระบบนี้ออกแบบให้สอดคล้องกับขั้นตอนดังกล่าว โดยเน้นการโอนเงินและแนบสลิป แล้วให้ผู้ดูแลตรวจสอบและอนุมัติเป็นหลัก

---

### บทที่ 3 วิธีดำเนินการ

**วิธีการพัฒนา**  
ใช้การพัฒนาระบบแบบเชิงวัตถุ (object-oriented) และออกแบบฐานข้อมูลจากความสัมพันธ์ของเอนทิตี้ (ER) และการไหลของข้อมูล (DFD) ตามที่ระบุในเอกสาร ER-DIAGRAM และ DFD-DIAGRAM ของโปรเจกต์

**เครื่องมือและเทคโนโลยี**  
- **ฝั่งคลายเอนต์และเซิร์ฟเวอร์:** Next.js 14 (App Router), React, TypeScript  
- **ฐานข้อมูล:** SQLite ผ่าน Prisma ORM  
- **การยืนยันตัวตน:** NextAuth.js (Credentials + JWT)  
- **การจัดรูปแบบและส่วนติดต่อผู้ใช้:** Tailwind CSS, shadcn/ui  
- **การจัดเก็บไฟล์:** เก็บไฟล์อัปโหลด (รูปห้อง สลิป) ในโฟลเดอร์ `public/uploads` และบันทึก path ลงฐานข้อมูล

**การดำเนินการ**  
ออกแบบโมเดลข้อมูล (User, Room, Booking, ServiceItem, HeroSlide) และ API สำหรับฝั่งลูกค้า (ลงทะเบียน ล็อกอิน จอง อัปโหลดสลิป ดูการจอง) และฝั่งแอดมิน (CRUD ห้อง จอง สินค้า/บริการ สไลด์ ผู้ใช้ และ API ตรวจสอบมัดจำ) จากนั้นพัฒนาหน้าเว็บให้เชื่อมกับ API และทดสอบการทำงานตามสถานการณ์การใช้งานหลัก

### บทที่ 4 ผลการดำเนินการ

**ระบบที่ได้**  
ระบบจองที่พักโฮมสเตย์ออนไลน์ทำงานบนเว็บได้ครบตามวัตถุประสงค์ โดยมีผลการดำเนินการสรุปดังนี้

- **ฝั่งผู้ใช้:** ลงทะเบียนและล็อกอินได้ ดูรายการห้องพัก รายละเอียดห้อง ราคาต่อคืน สินค้าและบริการ และสไลด์หน้าแรก เลือกวันเช็คอิน–เช็คเอาท์และจำนวนผู้เข้าพักแล้วระบบคำนวณราคารวม มัดจำ 50% และยอดคงเหลือ 50% แสดงบนฟอร์มจอง หน้ายืนยันการจอง และหน้ารายการจอง (ตรวจสอบการจองด้วยอีเมล) สามารถอัปโหลดสลิปมัดจำและดูสถานะการตรวจสอบ (รอตรวจ อนุมัติ ปฏิเสธ) ได้

- **ฝั่งผู้ดูแล:** ล็อกอินเข้าหน้าจัดการ (admin) จัดการห้องพัก (เพิ่ม/แก้ไข/ลบ และอัปโหลดรูป) จัดการรายการจอง (ดูรายการ กรอง แสดงสถานะมัดจำ) ตรวจสอบสลิปมัดจำ (อนุมัติ/ปฏิเสธและบันทึกหมายเหตุ) จัดการสินค้าและบริการ จัดการสไลด์หน้าแรก และจัดการบทบาทผู้ใช้ (เช่น เปลี่ยน role)

- **ข้อมูลและความปลอดภัย:** ข้อมูลจัดเก็บใน SQLite ผ่าน Prisma การเข้าถึง API ฝั่งแอดมินตรวจสอบบทบาท (ADMIN/EMPLOYEE) รหัสผ่านเก็บแบบเข้ารหัส (hash) และการอัปโหลดไฟล์จำกัดประเภทและขนาดตามที่กำหนด

### บทที่ 5 สรุปและข้อเสนอแนะ

**สรุป**  
โครงการพัฒนาระบบจองที่พักโฮมสเตย์ออนไลน์ได้ระบบเว็บแอปพลิเคชันที่รองรับการลงทะเบียน/ล็อกอิน การจองห้องด้วยการคำนวณมัดจำและยอดคงเหลือแบบ 50/50 การอัปโหลดสลิปมัดจำ และการตรวจสอบ/อนุมัติจากฝั่งผู้ดูแล รวมถึงการจัดการห้องพัก สินค้าและบริการ สไลด์หน้าแรก และผู้ใช้ ระบบใช้ Next.js, TypeScript, Prisma (SQLite) และ NextAuth ตรงกับวัตถุประสงค์และขอบเขตที่กำหนด

**ข้อเสนอแนะในการพัฒนาต่อ**  
1) เพิ่มการชำระเงินผ่าน payment gateway เพื่อรับชำระมัดจำหรือเต็มจำนวนออนไลน์ 2) เพิ่มการแจ้งเตือน (อีเมล/ไลน์) เมื่อมีการจองหรือเมื่อแอดมินอนุมัติ/ปฏิเสธสลิป 3) ขยายให้รองรับหลายที่พัก (multi-tenant) หรือหลายห้องพร้อมนโยบายยกเลิก/คืนเงิน 4) เพิ่มรายงานและสถิติการจอง (กราฟ รายเดือน/รายปี) สำหรับผู้ดูแล 5) ทดสอบความปลอดภัย (เช่น penetration test) และปรับปรุงการจัดการไฟล์อัปโหลดให้ปลอดภัยยิ่งขึ้น
